Compiling the Kernel

Contents

Compiling the kernel amounts to typing "make"

If you have multiple CPUs or CPU cores (grep processor /proc/cpuinfo), you can use the "-j" option of make For example, if you have 8 CPU cores: 


	make -j8
This will make things compiler faster. Some people even use 4*number-of-cpus as the -j argument.

If you use the -j argument, you may find that after an error, other jobs started concurrently by make continue to produce output, causing any diagnostics to scroll off the screen, and possibly causing to fail to realize that anything has gone wrong. After running make -j8, it's a good idea to check the status by: 


	# echo $?
	0
	#

The status should be zero.  If not, run make again, without
the -j option to see what the errors are (or, of course
you can try to scroll back in your terminal.)


Also, if you find yourself rebuilding the entire kernel
after running make clean, but
with the same .config file, 
ccache
can also help a lot to speed things up.
